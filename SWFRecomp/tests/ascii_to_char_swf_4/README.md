# ASCII_TO_CHAR Opcode Test (0x32)

This test verifies the implementation of the `ASCII_TO_CHAR` opcode (0x32) which converts an ASCII/Unicode code value to its corresponding character string.

## Opcode Details

- **Opcode**: `0x32` (ASCII_TO_CHAR)
- **Category**: String Operations
- **SWF Version**: 4+

## Operation

The opcode:
1. Pops a numeric value from the stack
2. Converts it to an integer (truncating decimals)
3. Converts the code point to a single-character string
4. Pushes the resulting string onto the stack

## Test Cases

The test includes 5 test cases:

1. **chr(65)** → Expected: `A`
   - Basic uppercase letter

2. **chr(97)** → Expected: `a`
   - Basic lowercase letter

3. **chr(33)** → Expected: `!`
   - Special character (exclamation mark)

4. **chr(32)** → Expected: ` ` (space)
   - Whitespace character

5. **chr(233)** → Expected: `é`
   - Extended ASCII character

## Expected Output

```
A
a
!

é
```

## Implementation Notes

- Values are masked to 0xFF (0-255 range) using bitwise AND
- Negative values wrap around due to the masking
- The implementation handles character encoding for extended ASCII (128-255)
